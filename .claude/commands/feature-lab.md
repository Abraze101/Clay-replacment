# Feature Lab

Read `CLAUDE.md` and the product documents under `docs/`.

This is a product-planning session. Do not implement application code, install dependencies, run migrations, call providers, or change an active implementation milestone.

When the user proposes a feature:

1. Clarify the user problem and target campaign.
2. Show how it fits Quick List, Call-Ready, Full Enrichment, or a new workflow.
3. Identify provider, storage, MCP/CLI, cost, privacy, compliance, and UI effects.
4. Recommend MVP, later milestone, or out of scope.
5. Draft explicit acceptance criteria.
6. Record the proposal under `docs/proposals/` only after the user approves the proposal text.

Canonical requirements and implementation milestones change only after the user approves the proposal and asks for it to be promoted.
