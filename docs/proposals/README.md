# Feature proposals

Use this folder for ideas that have not yet changed the accepted product scope.

Each proposal should contain:

- User problem and target user.
- Proposed behavior and workflow.
- Enrichment/profile impact.
- Provider and data requirements.
- CLI/MCP and web UI impact.
- Cost, privacy, compliance, and operational risks.
- Recommended milestone.
- Acceptance criteria.
- Decision status: proposed, accepted, rejected, or deferred.

Accepted proposals must be promoted into the canonical requirements, architecture, workflows, and implementation plan before code changes begin.
